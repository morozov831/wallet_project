# Wallet API

Асинхронный REST API сервис для управления электронными кошельками на FastAPI.

## О проекте

Сервис предоставляет возможность выполнять базовые финансовые операции: пополнение баланса (DEPOSIT) и снятие средств (WITHDRAW), а также просматривать текущий баланс. Реализована корректная работа в конкурентной среде — при параллельных запросах к одному кошельку используется пессимистическая блокировка на уровне базы данных (SELECT FOR UPDATE), что гарантирует целостность данных даже при высокой нагрузке. 

Приложение полностью асинхронное, построено на современном стеке FastAPI + SQLAlchemy 2.0 + asyncpg и упаковано в Docker для простоты развертывания. Миграции базы данных управляются через Alembic. Код покрыт тестами, включая сценарии конкурентных операций.

## Технологии

FastAPI, PostgreSQL, SQLAlchemy 2.0, Alembic, Pydantic, Docker, docker-compose, pytest, asyncpg.

## Тестовые данные

При запуске проекта (`docker compose up -d --build`) автоматически выполняются миграции и скрипт инициализации базы данных `seed.py`, который создает следующие тестовые кошельки:

| UUID | Баланс |
|------|--------|
| `f5db6a18-f29b-46e8-847b-b443a0250db1` | 1000.00 |
| `71767b2f-9ac0-403e-ab2c-eb6a6c72da87` | 500.50 |
| `9a5523d4-0346-4edc-90f4-b67eec63e434` | 10000.00 |
| `24a3860d-df1b-47fc-8d27-baf100dacb4e` | 0.00 |
| `4e8e2ca7-5a68-40fa-a568-44ccd3feff1b` | 999999.99 |

Эти UUID можно использовать для тестирования API без необходимости создавать кошельки вручную.

<h2>Установка и запуск</h2>
<h3>Установка:</h3>

1. Склонируйте репозиторий проекта из GitHub:
   ```bash
   git clone https://github.com/morozov831/wallet_project.git
   ```

2. Перейдите в каталог проекта:
   ```bash
   cd wallet_project
   ```
3. Создайте и активируйте виртуальное окружение, если оно не создано автоматически интерпретатором:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

4. Установите необходимые пакеты:
   ```bash
   pip install -r requirements.txt
   ```

5. Настройте файл окружения:
   ```bash
   cp .env.example .env
   ```
   
<h3>Запуск проекта</h3>
<ol>
    <li>Убедитесь, что Docker и Docker Compose установлены на вашей системе.</li>
    <li>Запустите следующую команду для сборки образов, создания и запуска контейнеров:
        <pre><code>docker compose up -d --build</code></pre>
    </li>
    <li>При запуске контейнеров автоматически выполняются миграции и скрипт создания тестовых данных <code>seed.py</code>.</li>
</ol>

<h4>Переход к документации Swagger</h4>
<p>Для взаимодействия с API перейдите по следующему адресу: 
    <a href="http://localhost:8000/docs">http://localhost:8000/docs</a>
</p>

<h3>Остановка проекта</h3>
<p>Чтобы остановить контейнеры Docker и удалить их, запустите следующую команду:</p>
<pre><code>docker compose down -v</code></pre>

<h3>Запуск тестов</h3>
<ol>
    <li>Запустите тестовую базу данных в Docker:
        <pre><code>docker compose -f docker-compose.test.yaml up -d</code></pre>
        <p><i>Будет запущен контейнер с PostgreSQL на порту 6000 для тестов.</i></p>
    </li>
    <li>Запустите тесты (локально):
        <pre><code>pytest -v</code></pre>
    </li>
    <li>После завершения тестов остановите тестовую БД:
        <pre><code>docker compose -f docker-compose.test.yaml down -v</code></pre>
    </li>
</ol>
